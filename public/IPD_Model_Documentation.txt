 1. Executive Summary

This document describes the design, implementation, and evaluation of a VMD-based Ensemble
Model for forecasting Brent Crude Oil prices. The system combines four complementary machine
learning models -- ARIMA, Mid-Frequency GRU, Sentiment-GRU, and XGBoost -- through a
stacking meta-learner to produce short-term oil price forecasts at 5, 7, and 14 trading day horizons.

The ensemble model with sentiment features achieved 77.3% directional accuracy and a mean
absolute price error of approximately $0.65 per barrel at the 5-day horizon -- outperforming all
individual baseline models across every evaluation metric. A key finding is that sentiment features
hurt individual model performance but consistently improve the full ensemble, demonstrating that
sentiment signals are only useful in combination with complementary price and trend signals.

 2. System Architecture

2.1 Overall Design

The system follows a multi-layer ensemble architecture built around Variational Mode
Decomposition (VMD). The raw Brent Oil log return series is first decomposed into three frequency
components -- trend, mid-frequency, and high-frequency -- each handled by a specialised model.
Their outputs are then combined by a stacking meta-model.

Layer           Model              Input                       Output
1 -- Trend      ARIMA                                          Trend log return forecast
2 -- Mid-Freq   GRU (price)        VMD trend component         Mid-freq return forecast

3 -- Sentiment  GRU (dual stream)  14 price features ×         Sentiment-aware forecast
4 -- High-Freq  XGBoost            LOOKBACK days               High-freq return forecast
5 -- Stacking   Ridge Regression                               Final ensemble forecast

2.2 VMD Decomposition

Variational Mode Decomposition (VMD) decomposes the log return series into K=3 modes with
parameters: alpha=2000 (bandwidth constraint), tau=0 (noise tolerance), DC=0, init=1 (uniform
initialisation), tol=1e-7. The three modes capture:

    · Mode 1 -- Trend component: slow-moving long-term price direction
    · Mode 2 -- Mid-frequency component: weekly to monthly cycles
    · Mode 3 -- High-frequency component: short-term noise and micro-structure

2.3 Stacking Meta-Model

A Ridge regression meta-model (alpha=1.0) is trained with walk-forward cross-validation (5 folds) to
optimally weight the four sub-model forecasts. One Ridge model is trained per horizon step --
meaning H5 has 5 Ridge models, H14 has 14 Ridge models. This direct multi-step approach avoids
error accumulation from recursive forecasting.

 4. Models Used

4.1 ARIMA -- Trend Component

    · Input: VMD trend_return series
    · Order selection: grid search over p  {0,1,2}, d  {0,1}, q  {0,1,2}
    · Selection criterion: validation-set RMSE using append() incremental updates
    · Inference: append() rolling forecast -- fits once, updates state cheaply per step
    · Purpose: captures long-term trend signal

4.2 Mid-Frequency GRU

    · Architecture: 1-layer GRU, hidden_size=64, dropout=0.3
    · Input shape: (LOOKBACK × 14 features)
    · Output: HORIZON predicted log returns (direct multi-step)
    · Training: Adam optimiser, lr=1e-3, early stopping (patience=10) on validation loss
    · Purpose: captures medium-term price cycles from pure price signals

4.3 Sentiment-GRU (Dual Stream with Attention)

    · Architecture: two parallel GRUs (price stream + sentiment stream) with attention mechanism
    · Price GRU: hidden_size=64, processes 14 price features
    · Sentiment GRU: hidden_size=64, processes 16 sentiment features with attention weighting
    · Fusion: concatenate price hidden state + attention-weighted sentiment hidden state
    · Output: HORIZON predicted log returns
    · Purpose: incorporates news sentiment to enhance price direction prediction

4.4 XGBoost High-Frequency Model

    · Architecture: one XGBRegressor per horizon step (direct multi-step)
    · Parameters: n_estimators=500, max_depth=3, learning_rate=0.05, subsample=0.8
    · Early stopping: rounds=20 on validation set -- prevents overfitting
    · Input: 8 high-frequency features (last single row, no lookback window)
    · Purpose: captures short-term micro-structure and high-frequency signals

4.5 Ridge Regression Meta-Model (Stacking Layer)

    · One Ridge model per horizon step (H5: 5 models, H14: 14 models)
    · Input: 4 features -- one forecast from each sub-model

 8. Sentiment Impact Analysis

The sentiment ablation was conducted across all three horizons, comparing models with and
without sentiment features. This is one of the most important findings of the study.

Experiment  Comparison             RMSE Change                 Verdict
H5          GRU alone              +1.04% worse                Sentiment HURTS
H5          Ensemble               -0.32% better               Sentiment HELPS
H7          GRU alone              +0.24% worse                Sentiment HURTS
H7          Ensemble               -0.40% better               Sentiment HELPS
H14         GRU alone              +1.81% worse                Sentiment HURTS
H14         Ensemble               -0.29% better               Sentiment HELPS

Key finding: Sentiment features consistently hurt GRU performance when used in isolation but
consistently help the full ensemble across all three horizons. This demonstrates that sentiment
signals are only useful when combined with complementary price and trend signals through the
stacking layer.

 11. How the System Works -- End to End

11.1 Training Pipeline

    · Step 1: Load raw price and sentiment CSV files
    · Step 2: Compute log returns, lag sentiment features by 1 day (no leakage)
    · Step 3: Engineer 30 features -- lags, volatility, RSI, momentum, sentiment EMAs
    · Step 4: VMD decomposition into trend, mid, high-frequency components
    · Step 5: Train ARIMA on trend component -- order selected by validation RMSE
    · Step 6: Train Mid-GRU on 14 price features with LOOKBACK-day sliding windows
    · Step 7: Train Sentiment-GRU on 30 features (dual stream + attention)
    · Step 8: Train XGBoost on 8 high-frequency features with early stopping
    · Step 9: Collect all sub-model forecasts on validation/test set
    · Step 10: Train Ridge meta-model with walk-forward cross-validation

11.2 Inference Pipeline (Production)

    · Step 1: Fetch yesterday's official closing price after market close (~5:30 PM ET)
    · Step 2: Build feature array from last LOOKBACK trading days of data
    · Step 3: ARIMA forecasts trend_return for next HORIZON steps
    · Step 4: Mid-GRU receives last LOOKBACK rows (21 days for H5) -- outputs 5 log returns
    · Step 5: Sentiment-GRU receives last LOOKBACK rows of price + sentiment -- outputs 5 log

         returns
    · Step 6: XGBoost receives last single row of HF features -- outputs 5 log returns
    · Step 7: Ridge meta-model combines 4 sub-model outputs into final ensemble forecast
    · Step 8: Convert log returns to prices: P_{t+h} = P_t × exp(sum of log returns to h)
    · Step 9: Build fan chart bands using historical error stds per horizon step
    · Step 10: Store one prediction record in database, serve to all users until next close

11.3 Fan Chart (Uncertainty Quantification)

Historical prediction errors at each horizon step are stored and used to compute uncertainty bands.
The bands are calculated as:

Band price = forecast_price × (1 + quantile)

where quantiles are computed from the distribution of relative errors: (actual - predicted) / predicted.
Three bands are shown: p10/p90 (80% interval), p25/p75 (50% interval), and p50 (median). Bands
widen with horizon as forecast uncertainty grows.
